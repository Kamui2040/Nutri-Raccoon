# Privacy model

## Local by default

Meal history, consumption amounts, dates, times, nutrition goals, body information, favourites, usage frequency, and custom notes remain local unless the user explicitly exports them.

## Optional network operations

Network use must be visible and configurable. Likely operations include barcode lookup, product updates, optional AI requests, and future community catalogue access.

## Community contribution boundary

A contribution may contain sanitised product information only. It must contain no:

- Username or account identifier
- Pseudonymous contributor identifier
- Device identifier
- IP-derived metadata stored in the public record
- Meal or consumption history
- Goals or body data
- Favourites or usage frequency
- Private notes
- Image metadata

The user must see an exact contribution preview and explicitly approve it.

## Images

Structured text contribution should precede image contribution. Any future image upload must crop or preview the intended image, strip metadata, explain licensing, and require separate consent.
