# Security policy

## Reporting

Do not disclose an unpatched vulnerability in a public issue. Use GitHub private vulnerability reporting when enabled, or the private contact method later published in this file.

## Sensitive information

Never include signing keys, credentials, API keys, private service endpoints, user databases, personal nutrition records, or unsanitised diagnostic material in a report.

## Supported versions

No public application version exists yet. A supported-version table will be added with the first release.
